# التحقق من بوابة المرحلة PH-03: العملاء والموردون

## أهداف المرحلة
- إدارة العملاء (قائمة، إنشاء، تعديل، حذف، عرض)
- إدارة الموردين (قائمة، إنشاء، تعديل، حذف، عرض)
- تحصيلات العملاء (قائمة، إنشاء، طباعة)
- أرصدة العملاء والموردين
- كشوف حسابات العملاء والموردين
- تقسيم العملاء إلى أقسام/فئات

## قائمة التحقق

### 1. وحدات API (Backend)

- [x] **GET /api/customers** — قائمة العملاء مع التصفية حسب الشركة
- [x] **POST /api/customers** — إنشاء عميل جديد
- [x] **PATCH /api/customers/[id]** — تعديل عميل
- [x] **DELETE /api/customers/[id]** — حذف عميل
- [x] **GET /api/customers/[id]** — عرض تفاصيل عميل
- [x] **GET /api/customers/[id]/receivables** — عرض مستحقات العميل
- [x] **GET /api/customers/[id]/statement** — كشف حساب العميل
- [x] **GET /api/suppliers** — قائمة الموردين مع التصفية حسب الشركة
- [x] **POST /api/suppliers** — إنشاء مورد جديد
- [x] **PATCH /api/suppliers/[id]** — تعديل مورد
- [x] **DELETE /api/suppliers/[id]** — حذف مورد
- [x] **GET /api/suppliers/[id]** — عرض تفاصيل مورد
- [x] **GET /api/suppliers/[id]/payables** — عرض مستحقات المورد
- [x] **GET /api/suppliers/[id]/statement** — كشف حساب المورد
- [x] **GET /api/customer-collections** — قائمة التحصيلات مع التصفية حسب الشركة
- [x] **POST /api/customer-collections** — إنشاء تحصيل جديد
- [x] **GET /api/customer-categories** — قائمة أقسام العملاء

### 2. واجهات المستخدم (UI)

- [x] **صفحة العملاء والموردين** — علامات تبويب للعملاء، الموردين، الأقسام
- [x] **صفحة تفاصيل العميل** — معلومات العميل + علامات تبويب المستحقات وكشف الحساب
- [x] **صفحة تفاصيل المورد** — معلومات المورد + علامات تبويب المستحقات وكشف الحساب
- [x] **صفحة التحصيلات** — قائمة التحصيلات مع زر طباعة
- [x] **صفحة إنشاء تحصيل** — نموذج إنشاء تحصيل جديد
- [x] **صفحة التحصيلات** — تصحيح حقل collectionNumber إلى id
- [x] **حوارات إنشاء/تعديل** — للعملاء، الموردين، الأقسام

### 3. الأمان والصلاحيات

- [x] **SUPPLIERS_VIEW** — صلاحية عرض الموردين
- [x] **SUPPLIERS_CREATE** — صلاحية إنشاء مورد
- [x] **SUPPLIERS_EDIT** — صلاحية تعديل مورد
- [x] **SUPPLIERS_DELETE** — صلاحية حذف مورد
- [x] **SUPPLIERS_PAYABLES_VIEW** — صلاحية عرض مستحقات المورد (جديد)
- [x] **SUPPLIERS_STATEMENT_VIEW** — صلاحية عرض كشف حساب المورد (جديد)
- [x] **COLLECTIONS_PRINT** — صلاحية طباعة التحصيل (جديد)
- [x] **CUSTOMERS_VIEW/CREATE/EDIT/DELETE** — صلاحيات العملاء
- [x] **CUSTOMERS_RECEIVABLES_VIEW** — صلاحية عرض مستحقات العميل
- [x] **CUSTOMERS_STATEMENT_VIEW** — صلاحية عرض كشف حساب العميل
- [x] **COLLECTIONS_VIEW/CREATE** — صلاحيات التحصيلات
- [x] **عزل الشركة (companyId)** — جميع وحدات API تتحقق من صلاحية الوصول للشركة
- [x] **requireDbPermission** — جميع وحدات API تستخدم التحقق الحديث من الصلاحيات

### 4. البيانات ونموذج Prisma

- [x] **حقل companyId في Supplier** — تمت إضافة علاقة الموردين مع الشركة
- [x] **@@unique([companyId, code])** — فهرس فريد مركب للموردين
- [x] **@@index([companyId])** — فهرس للتصفية حسب الشركة
- [x] **علاقة عكسية Company.suppliers** — ربط الشركة بالموردين

- تم إصلاح GAP B (صلاحيات مورد) — تحويل checkPermission → requireDbPermission وعزل الشركة
